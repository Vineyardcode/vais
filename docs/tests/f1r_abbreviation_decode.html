f1r_abbreviation_decode

scripts/f1r_abbreviation_decode.py

Abbreviation Decode Attempt on f1r

Applies JKP's paleographic glyph identifications (thread 2394) and
Cappelli abbreviation mappings to EVA-transcribed f1r text to see
if anything recognizable emerges.

Approach:
  1. Extract clean word tokens from f1r EVA transcription
  2. Apply positional substitution rules (JKP + Cappelli + Pelling)
  3. Show raw substitution output
  4. Try multiple language hypotheses (Latin, Italian)

Parameters

Edit a value to override it for an in-browser run (the script itself is never modified).

nametypevalue

Run it in your browser

Python + numpy on WebAssembly, entirely on your machine — nothing is uploaded. First run downloads the runtime (~15 MB) and the repository data pack (~9 MB); both are cached afterwards. Expect it to be slower than a native run.




Run it locally (full speed, byte-exact)


git clone https://github.com/Vineyardcode/vais
cd vais
pip install numpy flask
python scripts/f1r_abbreviation_decode.py        # this test; output also lands in results/
python webui/server.py          # or the full interactive UI at localhost:5000

ASSESSMENT
────────────────────────────────────────────────────────────────────────────────

This is a MECHANICAL substitution — it applies JKP's glyph
identifications as if they were a simple cipher key. This is
almost certainly an oversimplification because:
  1. The same EVA glyph may represent different abbreviations
     depending on context (e.g., EVA-y = -us/-um/-os/-con)
  2. Gallows are composite constructions, not single letters
  3. EVA character boundaries may be wrong (composite glyphs)
  4. There may be additional encoding layers (verbose cipher,
     alphabet substitution) on top of the abbreviation
  5. We don't know the underlying language

The purpose is to see if ANY recognizable patterns emerge,
not to produce a translation.
