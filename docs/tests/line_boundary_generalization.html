GENERALIZATION AND LINE BOUNDARY TESTS
===================================================

structure_cross_validation showed word-level bigrams don't generalize. Does the
morphological grammar generalize? Are lines syntactic units?

Sub-analyses:
  A) CROSS-VALIDATED MORPHOLOGICAL MI — Does suffix→prefix MI
       survive on held-out data?

  B) CROSS-VALIDATED CHARACTER MI — Does last_char→first_char MI
       generalize?

  C) LINE BOUNDARY TEST — Within-line vs cross-line transitions.

  D) JACKKNIFE STABILITY — Do key z-scores survive leave-k-out?

  E) THE qo-WORD NETWORK — Among -dy→qo transitions, is the
       specific qo-word predicted by the predecessor?

Parameters

Edit a value to override it for an in-browser run (the script itself is never modified).

nametypevalue
ALL_GALLOWSjson
SUFFIXESjson
GRAM_PREFIXESjson

Run it in your browser

Python + numpy on WebAssembly, entirely on your machine — nothing is uploaded. First run downloads the runtime (~15 MB) and the repository data pack (~9 MB); both are cached afterwards. Expect it to be slower than a native run.




Run it locally (full speed, byte-exact)


git clone https://github.com/Vineyardcode/vais
cd vais
pip install numpy flask
python scripts/line_boundary_generalization.py        # this test; output also lands in results/
python webui/server.py          # or the full interactive UI at localhost:5000
