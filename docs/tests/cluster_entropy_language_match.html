cluster_entropy_language_match

scripts/cluster_entropy_language_match.py

Cluster-Level h_char Test + Source Language Matching

KEY QUESTION: Does collapsing VMS chunks to their distributional
clusters (chunk_equivalence_classes, k=25) normalize the h_char anomaly?

If YES → homophones explain the anomaly, and we can match against
source languages at the functional character level.
If NO → the anomaly is structural, not just variant selection.

CRITICAL NULL MODEL: random clustering should NOT produce the same
uplift. If it does, the result is a mathematical artifact.

Parameters

Edit a value to override it for an in-browser run (the script itself is never modified).

Run it in your browser

Python + numpy on WebAssembly, entirely on your machine — nothing is uploaded. First run downloads the runtime (~15 MB) and the repository data pack (~9 MB); both are cached afterwards. Expect it to be slower than a native run.




Run it locally (full speed, byte-exact)


git clone https://github.com/Vineyardcode/vais
cd vais
pip install numpy flask
python scripts/cluster_entropy_language_match.py        # this test; output also lands in results/
python webui/server.py          # or the full interactive UI at localhost:5000
